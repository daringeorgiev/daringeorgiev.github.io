# daringeorgiev.github.io

Personal portfolio site for Darin Georgiev. Single-page HTML site deployed on GitHub Pages from the `master` branch.

## Standards & quality bar

Every change to this project must maintain or improve the following:

- **Lighthouse scores** — target 90+ in all four categories (Performance, Accessibility, Best Practices, SEO). Run Lighthouse in Chrome DevTools before considering any task done.
- **W3C validity** — HTML must pass the [W3C Markup Validator](https://validator.w3.org/) with no errors. Warnings are acceptable if they are justified.
- **Accessibility** — semantic HTML elements, ARIA labels on interactive controls, `alt` text on all images, sufficient color contrast. Never remove an accessibility attribute without a replacement.
- **Performance** — do not regress on lazy loading, critical CSS inlining, minified assets, or WebP image delivery.
- **Responsiveness** — layout must work correctly at mobile (360px+), tablet (768px+), and desktop (1200px+) widths. Test every change at all three breakpoints.
- **Browser compatibility** — must work on the last 2 versions of Chrome, Firefox, Safari, and Edge, and on iOS Safari and Chrome for Android.

## Build

```bash
# Compile SCSS → css/styles.css + css/styles-min.css
gulp css

# Minify JS → js/scripts-min.js (and googleAnalytics-min.js, serviceworker-cache-polyfill-min.js)
gulp scripts

# Extract and inline critical CSS into dist/index.html
gulp critical
```

`gulp critical` reads `css/styles-min.css` and writes an updated `index.html` into `dist/` with critical styles inlined in the `<head>`. Run it after any significant CSS change that affects above-the-fold rendering.

## Stack

### Current state

- **HTML** — single `index.html`, no build step for markup
- **CSS** — SCSS compiled by `gulp-sass`. Entry point: `css/styles.scss`. Bootstrap 3.4.1 + Bootswatch Cosmo compiled into `css/styles-min.css` via SCSS imports from `libs/bootstrap-sass/`.
- **JS** — jQuery 3.5.1 for all custom behavior. Bootstrap 3.4.1 JS bundle for navbar collapse, scrollspy, carousel, and affix (sticky navbar).
- **PWA** — `serviceWorker.js` caches key assets for offline use; `manifest.json` defines the app shell.

> Reflects the state after the Bootstrap 5 migration — see Planned work.

## Key patterns

> **Note:** the sticky navbar pattern below reflects the target state after the Bootstrap 5 migration.

### Sticky navbar (affix replacement)
Bootstrap 3's `affix` plugin was removed in Bootstrap 5. The sticky behaviour is now handled by a scroll listener in `js/scripts.js` that toggles `.navbar--scrolled` on the `<nav>` at `scrollY > 450`. The corresponding styles live in `css/styles.scss` under `.navbar--scrolled` (darker background + box-shadow).

### Critical CSS
The `<style>` block inside `<head>` in `index.html` is auto-generated by `gulp critical` — do not edit it by hand. Re-run `gulp critical` after CSS changes to keep it in sync.

### Images
Profile picture uses a `<picture>` element with WebP + JPG fallback. All other images use `data-src` for lazy loading, swapped to `src` by `js/scripts.js` on window load.

## Visual improvement suggestions

Not all of these need to be implemented — pick what fits. Each is independent and can be done in any order after the planned work above.

- **Scroll-triggered entrance animations** — sections fade/slide in as they enter the viewport using `IntersectionObserver`. No library needed. High impact, low effort.
- **Skills as tags** — replace the nested `<ul>` with pill-shaped tags grouped by category (Languages, Frameworks, Tools). More scannable and visually modern.
- **Project cards with screenshots** — add a small thumbnail to each project block. Even a simple image makes the section visually distinctive and easier to scan.
- **Experience/Education timeline** — a vertical line with dots to mark each entry. Makes the chronology faster to read than plain headings.
- **Dark mode** — add a `prefers-color-scheme: dark` media query using CSS custom properties for colors. Works alongside the existing palette without a full redesign.
- **"Currently working with" callout** — a small highlighted note in the About or Skills section showing the stack you're actively using. Signals to recruiters that the page is maintained.
